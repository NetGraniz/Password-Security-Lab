# Password Security Lab

Password Security Lab — статическое веб-приложение для анализа стойкости паролей, оценки энтропии, проверки известных утечек и генерации безопасных паролей, парольных фраз и ключей.

Проект работает на HTML5, CSS3 и Vanilla JavaScript. Backend, база данных, авторизация и аналитика не используются, поэтому приложение подходит для публикации на GitHub Pages.

## Возможности

- Анализ длины, используемых наборов символов, теоретической энтропии и практической оценки.
- Итоговый вердикт: «Не использовать», «Слабый», «Нормальный», «Хороший», «Отличный».
- Современные рекомендации с приоритетом длины, уникальности, отсутствия словарных слов, шаблонов, дат и утечек.
- Проверка пароля по базе Have I Been Pwned Pwned Passwords через k-anonymity.
- Оценка примерного времени полного перебора для разных сценариев атакующего.
- Интерактивный график практической стойкости.
- Генератор случайных паролей с Web Crypto API и `randomInt` без modulo bias.
- Гарантированное включение выбранных типов символов в случайный пароль.
- Режим парольной фразы с английским и русским словарём.
- Генератор запоминаемых паролей, лицензионных ключей и читаемых ключей.
- Приватный режим, включённый по умолчанию.
- Безопасная история: проверки сохраняются только как технические данные без исходного пароля.
- Настройка сохранения сгенерированных значений в историю, по умолчанию выключена.
- Избранное только после явного подтверждения пользователя.
- Автоочистка сгенерированных значений через 60 секунд.
- Экспорт истории в TXT и CSV с предупреждением о чувствительных данных.
- Светлая, тёмная и автоматическая тема.
- Адаптивный интерфейс для desktop и mobile.

## Безопасность и приватность

Все вычисления выполняются локально в браузере пользователя. У приложения нет серверной части, оно не собирает аналитику и не передаёт пароли владельцу сайта.

Введённые в анализатор пароли не сохраняются в LocalStorage. Если пользователь явно сохраняет результат проверки, сохраняются только технические данные: дата, длина, уровень, теоретическая энтропия, практическая оценка и итоговый вердикт. Сам пароль, его фрагменты и исходное значение не сохраняются.

Приватный режим включён по умолчанию. В этом режиме приложение не сохраняет результаты проверок, сгенерированные значения, избранное и пользовательские шаблоны.

Проверка утечек использует Have I Been Pwned Pwned Passwords API по схеме k-anonymity:

1. SHA-1 хэш пароля вычисляется локально в браузере.
2. На API отправляются только первые 5 символов хэша.
3. Полный пароль и полный SHA-1 хэш не отправляются.
4. Сравнение оставшейся части хэша выполняется локально.

Важно: не вводите реальные пароли от важных аккаунтов без необходимости. Даже локальные инструменты лучше использовать осторожно, особенно на чужих устройствах.

## Проверка утечек

Кнопка «Проверить в утечках» не запускается автоматически при вводе символов. Проверка выполняется только после явного действия пользователя. Если API временно недоступен, приложение показывает аккуратное сообщение и анализатор продолжает работать.

## Запуск локально

Откройте `index.html` в современном браузере. Для GitHub Pages достаточно разместить файлы в корне репозитория:

- `index.html`
- `styles.css`
- `words.js`
- `script.js`

## Ограничения

- Проект не использует backend, базы данных, авторизацию и аналитику.
- История хранится только локально в LocalStorage.
- Проверка утечек требует доступа браузера к `https://api.pwnedpasswords.com`.
- Оценка стойкости является приблизительной и не заменяет полноценный аудит безопасности.
